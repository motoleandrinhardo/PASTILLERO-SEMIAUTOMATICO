2. Objetos

LiquidCrystal_I2C lcd(0x27, 16, 2);

Se crean:

rtc → para acceder al reloj.
lcd → pantalla LCD dirección I2C 0x27, tamaño 16x2.
3. Definición de pines
Sensores
#define CNY_LUNES      36
#define CNY_MARTES     39
#define CNY_MIERCOLES  34
#define CNY_JUEVES     35
#define CNY_VIERNES    32
#define CNY_SABADO     33
#define CNY_DOMINGO    27

Hay un sensor para cada compartimento de pastillas.

Por ejemplo:

Lunes → GPIO 36
Martes → GPIO 39
etc.

La idea es que cada sensor detecte si la pastilla sigue presente o si se retiró.

Indicadores
#define LED_VERDE      19
#define LED_ROJO       18
#define BUZZER         23
LED verde → pastilla tomada.
LED rojo → pendiente.
Buzzer → alarma sonora.
4. Configuración principal
const int ALARM_HOUR = 10;
const int ALARM_MINUTE = 35;

La alarma se activará todos los días a:

10:35

const unsigned long BUZZER_TIME = 90000;

90.000 ms = 90 segundos = 1,5 minutos.

const int UMBRAL_APERTURA = 300;

Valor límite del sensor.

Si la lectura baja de 300:

lectura < 300

el sistema interpreta que la pastilla fue retirada.

15. Manejo del buzzer

Mientras:

PASTILLA TOMADA
Resumen general del funcionamiento

Cada día el sistema:

Detecta qué día es mediante el RTC.
Observa únicamente el sensor correspondiente a ese día.
Muestra hora y estado en la LCD.
A las 10:35 activa una alarma si la pastilla no fue retirada.
Si el sensor detecta que la pastilla fue sacada:
marca "tomada",
apaga la alarma,
enciende LED verde.
A medianoche reinicia el estado para el día siguiente.

En forma de flujo:

Nuevo día
    ↓
Pastilla pendiente
    ↓
10:35
    ↓
Suena buzzer
    ↓
Usuario retira pastilla
    ↓
Sensor detecta cambio
    ↓
LED verde
    ↓
Buzzer apagado
    ↓
Esperar próximo día
